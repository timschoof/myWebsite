Useful resources for reproducible workflow in R

8 April 2019 2 min read

Notes and talk about making data analysis more reproducible https://github.com/karthik/rstudio2019

Creating publication ready articles in R https://github.com/crsh/papaja

Welcome to the future of paper writing!! Ok, I’ll tone down my enthusiasm, but… it’s AWESOME!!!

You can write papers in R and it’s super neat. So reproducible!

I’m sure there are lots of different ways to go about this successfully, but here’s what’s working well for me. I’ve started using the papaja (Preparing APA Journal Articles) R package. It’s basically a template to produce documents that follow APA manuscript guidelines. In other words, you can write something in R that’s publication ready!

Getting started

First, make sure you have the latest versions of R and RStudio installed – always a good idea.

Next, you need to install the package by typing this in RStudio:

# Install devtools package if necessary
if(!"devtools" %in% rownames(installed.packages())) install.packages("devtools")

# Install the stable development verions from GitHub
devtools::install_github("crsh/papaja")

Now, what you’ll see is that it’s asking you if you want to update / install a bunch of packages. You might say yes, you might say no. In either case, for me it didn’t do anything. Instead, I ended up making a note of all the packages it mentioned and installed them by hand: install.packages(“whateverPackage”) I then tried again. Eventually, it stopped suggesting packages I should update and it worked.

Yay, you managed to do the hardest bit, you installed the package!

Writing

Now, just open paper.rmd and start typing :-) To turn it into a pdf / LaTeX etc. file, just click on ‘knit’.

Can also pull from other R scripts.

Version control

To be extra cool, put everything in a github repository! No, seriously, not just for the street cred, it’s actually useful. You can have them set to private if you’re at an educational institution.
